# 小聚记账 (Web版) 产品需求文档 (PRD)

## 1. 产品概述
“小聚记账”是一款专为熟人聚会、桌游娱乐等线下场景打造的 Web 端记录工具。旨在帮助用户在各类活动中便捷地记录、统计各项物品数量（如饮料瓶数、游戏积分等）或个人欠数，并提供基础的利息/增量推算及数据同步功能。
产品采用标准 Web 架构与**全平台响应式设计**，能够完美自适应 PC 桌面、iPad 平板及各类手机移动端浏览器的屏幕尺寸，后端部署于 Google Cloud 平台。

## 2. 核心用户角色与权限

系统依托标准的用户账户体系（如邮箱密码或第三方 OAuth 授权）进行身份认证，包含三种核心群组角色：
1. **群主 (Creator)**：群组的创建者，拥有最高权限。
2. **子管理员 (Sub-Admin)**：由群主任命，协助管理账目。
3. **普通群友 (Member)**：通过邀请链接加入群组，认领身份卡片后可参与记账。

| 功能权限 | 群主 | 子管理员 | 普通群友 | 游客 (未认领) |

| 给他人“记一笔” / 核销 | ✅ | ✅ | ❌ | ❌ |
| 添加新成员卡片 | ✅ | ✅ | ❌ | ❌ |
| 修改成员备注名 | ✅ | ✅ | ❌ | ❌ |
| 强制调平账目 | ✅ | ✅ | ❌ | ❌ |
| 认领空闲成员身份 | ✅ (可绑自己) | ❌ (已绑) | ❌ (已绑) | ✅ |
| 设置/取消子管理员 | ✅ | ❌ | ❌ | ❌ |

## 3. 核心功能模块

### 3.1 账号与身份模块
- **注册与登录**：支持通过邮箱+验证码、密码，或使用 Google 等第三方账号一键授权登录。
- **个人资料**：用户可进入个人中心，自行上传系统头像并设置自定义昵称。

### 3.2 群组大厅模块
- **我的群组列表**：以列表形式清晰展示用户创建的、以及作为成员加入的所有群组。
- **创建群组**：输入群组名称即可快速创建。创建者默认成为该群的“群主”。
- **加入群组**：支持通过输入群组邀请码加入，或直接点击他人分享的专属邀请链接加入。

### 3.3 群组详情模块
- **群组信息看板**：头部展示群组名称、当前群组总计数量及自定义单位。
- **成员列表**：
  - 展示所有成员卡片，包含头像、备注名、真实昵称、身份角标、当前记录数量。
  - **排序功能**：支持按“默认顺序(添加时间)”、“按姓名首字母”、“按数量(从多到少)”进行动态排序。排序偏好需保存在用户浏览器的 localStorage 中。
- **身份认领体系 (核心链路)**：
  - 管理员点击 `[+ 添加]` 生成一个未激活的“虚拟成员卡片”（仅输入备注名）。
  - 新用户通过链接进入群组后，点击未绑定的空白卡片，可将其与自己的 Web 账户深度绑定（认领身份）。
  - 认领完成后，该卡片将自动同步展示该真实用户的 Web 头像和昵称。
- **快速交互**：为提升聚会时的操作效率，在成员数量旁提供 `[+1]` 快速按钮，点击即可快速将数量加 1。

### 3.4 账目流水模块
- **记账操作**：
  - **增加数量 (记一笔)**：输入正数，增加当前余额。
  - **核销数量**：输入正数，减少当前余额（仅管理员可用）。
  - **强制调账**：直接输入最终的数值结果进行覆盖（仅管理员可用）。
- **流水明细**：在每个成员的详情页下方，按时间倒序展示该成员的所有变动流水（包含操作类型、变动数额、操作时间及备注）。

### 3.5 设置中心模块 (仅群主可见)
- **基础设置**：
  - 自由修改账目单位（如：个、瓶、杯、分等）。
- **计息/增量推演设置**：
  - **默认利率**：设置百分比数值。
  - **计算类型**：支持“单利（仅按本金计算）”与“复利（利滚利）”。
  - **计算频率**：支持 每日一次、每周一次、每月一次、每季度一次、每半年一次、每年一次。
- **结果预览**：在设置页实时展示基于当前配置的趋势预览。以“10个单位”为基准，自动推演并展示未来 6 个计算周期的具体日期和结果。
- **子管理员管理**：查看、管理当前子管理员列表。
- **危险操作**：解散群组（需弹窗二次强力确认，操作后清空所有关联成员及流水数据）。

## 4. 界面与交互规范 (跨端自适应)
为了保证良好的跨端体验，前端界面必须严格遵循**响应式设计 (Responsive Web Design)** 原则：
- **移动端 (Mobile)**：采用单列流式布局，优化大拇指触控热区（如大按钮、底部弹窗 ActionSheet），保证单手操作效率。
- **平板端 (iPad/Tablet)**：利用额外的屏幕宽度，采用双列或多列卡片布局，如“群组信息看板”与“成员列表”可以左右分栏显示，减少屏幕空间的浪费。
- **桌面端 (PC)**：引入全局侧边导航栏（Sidebar）与主内容区（Main Area）的经典管理后台布局，利用鼠标悬停（Hover）特效增强交互反馈，列表展示可利用更宽的表格或多列网格形态。

## 5. 技术架构建议 (Google Cloud 部署)
1. **前端架构**：推荐使用 React / Vue3 (如 Next.js 或 Vite) 配合 Tailwind CSS 或其他响应式 UI 库（如 MUI、Element Plus）构建，原生支持媒体查询（Media Queries）以适配各级断点。
2. **后端体系**：推荐使用 Node.js 构建 RESTful / GraphQL API 部署在 Google Cloud Run，或直接使用 Firebase (包含 Authentication 和 Firestore) 进行快速构建。
3. **数据库模型**：
   - `Users`: 存储用户核心信息（邮箱、密码哈希、头像、昵称等）。
   - `Groups`: 存储群组信息及配置。
   - `Members`: 关联群组与真实用户（记录当前余额与角色权限）。
   - `Records`: 存储所有数量变动的操作流水。
4. **鉴权机制**：采用标准的 JWT (JSON Web Token) 进行会话维持，通过路由鉴权拦截越权操作。
5. **社交裂变**：采用带参数的 URL 路由转发进行邀请分享，例如 `https://domain.com/group/:id?inviteCode=xxx`，配合 Web Clipboard API 实现一键复制分享链接。

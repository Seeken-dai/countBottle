# CountBottle 小聚记账 - 产品需求文档 (PRD)

## 1. 产品简介
CountBottle (小聚记账) 是一款专为朋友聚会、合租室友、牌局等小团体设计的轻量级在线记账工具。无需下载 App，通过微信或浏览器打开即可实时同步账单数据。

## 2. 核心功能设计

### 2.1 用户与认证系统
- 支持使用邮箱+密码进行注册和登录。
- 支持通过 Google 账号一键授权登录（待配置第三方 OAuth）。
- 拥有个人主页，可修改头像和昵称。

### 2.2 群组系统 (Group)
- 用户可以自由创建“记账群组”。
- 每个群组包含独立的计价单位（如：元、瓶、积分、美元）。
- 群主可以为群组设置“群公告”和“计息规则”。
- **分享机制**：一键复制邀请链接，新用户点击链接即可自动加入该群组。

### 2.3 成员与账单系统 (Members & Records)
- **卡片式成员列表**：群组内每一位成员都有一张独立的“成员卡片”，实时显示当前余额。
- **快捷记账**：点击成员卡片，弹出记账面板。
  - **加减账**：输入数字，点击“+”或“-”，直接增减该成员的余额。
  - **覆盖记账**：输入数字，点击“=”，强制将该成员的余额修改为指定数值（常用于结账清零）。
- **实时同步**：基于 Firebase Firestore 的 WebSocket 技术，任何人的记账操作都会在 1 秒内同步到所有正在看这个页面的成员屏幕上。
- **流水记录**：在卡片背面（或详情页）可以看到该成员的历史账单流水，明确“谁在什么时候给 TA 加/减了多少钱”。

### 2.4 自动化计息系统 (Interest System)
- **灵活配置**：群主可以设置计息方式（单利、复利），计息周期（每日、每周、每月、每年）以及利率。
- **自动结息机制**：
  - 基于用户的浏览行为触发被动计算（懒执行策略 Lazy Evaluation），极大节省服务器定时任务成本。
  - 只有当成员的余额为**负数（欠款）**时，系统才会按照设定的利率自动产生利息并入账单。

### 2.5 深色模式 (Dark Mode)
- 适配系统级别的深色模式，也可手动切换，保护深夜记账时的视力。

## 3. 技术架构设计
- **前端框架**：Next.js (App Router), React 18, TailwindCSS, Framer Motion
- **部署架构**：静态页面单页应用 (SPA) 架构
- **云服务引擎**：Google Firebase
  - *Hosting*：用于承载所有的静态网页内容。
  - *Firestore*：用于存储核心业务数据，提供实时监听能力。
  - *Authentication*：用于处理账户体系安全。

# CountBottle 小聚记账 - 运维与架构手册

## 1. 服务器域名与服务管理在哪里？

本系统目前托管于 **Google Firebase** 云平台。

- **域名查看与管理**：
  打开 [Firebase 控制台 (Firebase Console)](https://console.firebase.google.com/)，选择你的项目 `countbottle-web`。
  在左侧菜单栏点击 **Build (构建)** -> **Hosting (托管)**。
  在这里你可以看到：
  1. 系统为你免费分配的默认域名（通常是 `countbottle-web.web.app` 和 `countbottle-web.firebaseapp.com`）。
  2. 部署历史记录（每一次 GitHub Actions 的推送都会在这里生成一条版本记录，如果某次部署出了 Bug，你可以在这里点击“一键回滚”）。
  3. 自定义域名（如果你以后买了 `.com` 域名，可以在这里绑定，免费且自动带 HTTPS 证书）。

- **数据库管理**：
  在左侧菜单栏点击 **Build** -> **Firestore Database**。
  这里是你的核心数据中心，你可以在里面直观地看到 `Groups`, `Members`, `Records`, `Users` 等数据表，甚至可以直接在网页上手动修改数据。

## 2. 为什么在 Google Cloud Run 里看不到这个服务？服务到底放在哪了？

**之前看不到的原因：**
由于之前的架构限制，部署一直失败，代码根本没有推送到 Cloud Run，自然在里面找不到。

**现在的真实情况（非常重要）：**
经过我们刚刚的**架构大重构（降级为纯前端静态 SPA 模式）**，我们的系统已经**彻底脱离了 Cloud Run（动态服务器）！**

你的系统现在不再需要传统的“后端服务器”来运行。打包出来的纯静态 HTML/JS 代码，被 Firebase 直接分发到了 **Firebase Hosting 的全球 CDN（内容分发网络）节点**上。
- **优势**：没有服务器闲置成本、无需唤醒时间（极速加载）、**永远免费（不需要绑卡）**！
- 数据交互全部由浏览器直接和 Firebase Firestore（数据库）通过加密的长连接完成。

## 3. Firebase 的免费额度介绍 (Spark 计划)

目前系统处于 Firebase 的完全免费层（Spark 计划），不绑定信用卡也能永久使用。对于你的小团队记账需求，这个额度用一辈子都用不完。

### Hosting (网页托管)
- **存储空间**：10 GB（目前咱们代码包才几 MB，九牛一毛）
- **数据传输流量**：10 GB / 月（纯文字加代码，供几百人天天用也用不完）

### Firestore (数据库)
- **存储空间**：1 GB 免费（只存文字账单记录，1 GB 能存几千万条）
- **数据库读取 (Reads)**：50,000 次 / 每天（核心限制）
  - *系统设计了快照监听，只要数据没变，只消耗极少读取量。只有团队规模非常大且操作极度频繁时才可能触及。*
- **数据库写入 (Writes)**：20,000 次 / 每天（每天可以记 2 万笔账，完全够用）
- **数据库删除 (Deletes)**：20,000 次 / 每天

### Authentication (用户认证)
- **邮箱/密码登录**：每天 50,000 次活跃用户认证（无限量添加新用户）。
- **Google 快捷登录**：无限免费。

> **💡 运维建议：**
> 如果未来系统用的人实在太多（比如超过了几万个群组），导致数据库读取量超过了每天 5 万次，你只需要在 Firebase 控制台里点击升级为 Blaze 计划（绑定信用卡）。
> 哪怕升级了 Blaze 计划，只要不超过免费额度，依然是 **0 元**。超出的部分按次计费（非常便宜，每 10 万次读取大概才几毛钱人民币）。
